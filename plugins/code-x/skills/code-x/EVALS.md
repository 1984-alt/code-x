# Code-X V1 — EVALS (protocol self-tests / poka-yoke)

> Tiny tests that prove the PROTOCOL still enforces its own rules. Part of Kaizen (the verification sensor). These are NOT app tests. Run RARELY — never per module.

## When to run
On a protocol change · on V1 first creation · before a project's first build under V1 · before declaring V1 stable. **Not** on every module build.

## The set (45 — grows slowly; all locked)

- **EVAL-002** — a Mode A card FAILS if it touches backend/database files.

## Growth rule (Kaizen / anti-bloat)
The set grows **slowly** — one new eval per **new failure-class** learned (NOT one per lesson). Each eval stays tiny. If evals grow heavy, **consolidate**. The anti-bloat ceiling applies here too.

> A new failure that slips through becomes: postmortem → lesson → (if it's a new class) a new eval so it can never recur.
