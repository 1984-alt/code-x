# Code-X V1 — EVALS (protocol self-tests / poka-yoke)

> Tiny tests that prove the PROTOCOL still enforces its own rules. Part of Kaizen (the verification sensor). These are NOT app tests. Run RARELY — never per module.

## When to run
On a protocol change · on V1 first creation · before a project's first build under V1 · before declaring V1 stable. **Not** on every module build.

## The set (25 — grows slowly)
- **EVAL-001** — final-ready FAILS if any P0/P1/P2/P3 > 0.
- **EVAL-002** — a Mode A card FAILS if it touches backend/database files.
- **EVAL-003** — a builder card FAILS if it changes a `forbidden_files` entry.
- **EVAL-004** — a STOP card FAILS if it asks the CEO to debug / read a stack trace.
- **EVAL-005** — the Card Compilation Gate FAILS if a required invariant is missing from a card.
- **EVAL-006** — a build card FAILS the Card Compilation Gate if its `source_map` is missing or doesn't trace to a frozen packet hash, OR if `cross_review.family == executor.family` without an authorised `family_substituted` + `ceo_authorization_ref`; the authorised stand-in opens a `CROSS_FAMILY_RECHECK_PENDING` finding that blocks final-ready. *(added: LESSON-CARD-001)*
- **EVAL-007** — a card or packet bound for a cloud reviewer FAILS the cloud-safe scan if it still contains real private labels / amounts. *(added: LESSON-PRIVACY-001)*
- **EVAL-008** — a protocol-file edit that re-states a registered rule with divergent wording FAILS `cx check consistency` (the drift lint IS this eval's mechanical body). *(added: PROP-001/PROP-002, LESSON-DRIFT-001)*
- **EVAL-009** — the contract-bite harness: every `cx` gate clause carries a pinned BAD fixture the binary must REJECT at the right severity + a good fixture it must PASS; corrupting any clause turns `run_contracts.py` red (meta-test). A green check may only ever mean the gate FIRED. *(added: PROP-004, LESSON-ENFORCE-001)*
- **EVAL-010** — a review-dispatch card (`review_dispatch.required: yes`, or any mode `REVIEW`/`FINAL_READY` card) FAILS `cx check card` if any three-leg-ask field (`continuity`/`problems`/`approach_improvement`) is missing, a placeholder (`asked`/`yes`/`included`/`N/A`/blank), or lacks both its canonical label and a `prompt_ref` — a review dispatched without all three legs must never read green. Mechanical body = the 5 `CARD-REVIEW-DISPATCH-*` contract clauses. *(added: PROP-015, LESSON-REVIEW-ASK-001)*
- **EVAL-011** — a PLANNING-mode session FAILS session-start if `session_start.lessons_preload` is missing or lacks status/file/hash/read_by/timestamp/active_lesson_count. The ack ONLY — never "did the model understand the lessons" (stays deterministic). *(added: PROP-017, LESSON-REVIEW-HARVEST-001; mechanical body = the STATE-LESSONS-PRELOAD-ACK clause, BUILT + biting 2026-06-11)*

- **EVAL-012** — a BUILD-mode session FAILS if a build turn closes without its rail receipts: missing/stale `protocol_boot_ack` (machine-generated by `cx check boot`, never model-authored) fails session-start; a card claiming done without a `build-turn` receipt, or a turn ending without a `close-turn` receipt (typed `findings_delta` reconciled with state `open_findings`; `vault_sync` = PASS | SKIPPED_WITH_REASON | NOT_APPLICABLE with reason), FAILS close. *(added: PROP-018, LESSON-ENGINE-RAILS-001; mechanical body BUILT + biting 2026-06-12 = STATE-BOOT-ACK-REQUIRED/STALE · BOOT-FAILS-ON-BAD-STATE · BUILD-TURN-NO-TEST-COMMAND/TEST-FAILS · CLOSE-TURN-TYPED-DELTA-REQUIRED/DELTA-STATE-MISMATCH/VAULT-SYNC-ENUM · EVALS-SUITE-FAILURE-BITES)*
- **EVAL-013** — a card that touches templates/static/HTML-rendering routes or deletes Mode A fixture files FAILS `cx check card` without `ui_contract` locked artifacts in its read set; an engine wave purging a Mode A screen FAILS G3 without a `fixture_replacement_map` row; a UI module FAILS design-fidelity when the live DOM/markers miss the locked `ui_marker_manifest` (screenshot = supporting proof only). *(added: PROP-019, LESSON-UI-CONTRACT-CARRY-001; mechanical body BUILT + biting 2026-06-12 = CARD-UI-CONTRACT-REQUIRED/IN-READ-SET · CARD-FIXTURE-REPLACEMENT-MAP · CARD-VISUAL-CONTRACT-CONTEXT · DESIGN-FIDELITY-MARKER-MISSING/SCREENSHOT-REQUIRED)*
- **EVAL-014** — `cx check state` FAILS when `cross_family_boundary: ceo_deferred` lacks `ceo_decision_ref` + `deferred_review_blocks`, when `review_boundary` fields are missing/non-enum, or when a state with status `PROTOCOL_INCIDENT` opens a new build card / claims module done / requests cross-family / goes final-ready before the incident records cause + repair. *(added: PROP-020, LESSON-DEVIATION-TYPED-001; mechanical body BUILT + biting 2026-06-12 = STATE-REVIEW-BOUNDARY-REQUIRED/ENUM · STATE-CEO-DEFERRED-NEEDS-REF · STATE-DEVIATION-TYPED · STATE-INCIDENT-CAUSE-REPAIR · CARD-INCIDENT-BLOCKS-NEW-BUILD/CROSS-FAMILY · FINAL-READY-BLOCKED-BY-INCIDENT)*

## Growth rule (Kaizen / anti-bloat)
The set grows **slowly** — one new eval per **new failure-class** learned (NOT one per lesson). Each eval stays tiny. If evals grow heavy, **consolidate**. The anti-bloat ceiling applies here too.

> A new failure that slips through becomes: postmortem → lesson → (if it's a new class) a new eval so it can never recur.
- **EVAL-015** — same-tier fix re-dispatch bites: `fix_cycles` attempt-2 at the same seat tier after a failed fix-1 FAILS; attempt-3 not above attempt-2 FAILS; missing `finding_id`/`source_review_ref` FAILS; a valid escalated path (fix-1 same seat mechanical → fix-2 one tier up) PASSES. *(added: PROP-021, LESSON-FIX-ESCALATION-001; mechanical body = v1.09 TO_BUILD remainder)*
- **EVAL-016** — proof-card self-grading bites: claimed PASS whose log lacks expect_contains/contains a fail_marker = P0; acceptance-proof claim with no demonstrated red run FAILS; interaction proof with dispatched event family ≠ handler family = P1; baseline that is builder-produced same-wave output = P0; build translation key/state absent from `ui_lock_manifest` = P1; manifest without `audit_status: PASS` + `ceo_acceptance_ref` blocks build. *(added: PROP-022, LESSON-PROOF-HONESTY-001; mechanical body = v1.09 TO_BUILD remainder)*
- **EVAL-017** — the V1.10 module_acceptance gate family bites: a module-advancing card whose earlier modules are not yet accepted FAILS `cx check module-start` (P0); a hand-edited `accepted` with no bound receipt, a mismatched hash, or a non-`accepted` verdict FAILS `cx check module-acceptance` (P0); `cx check final-ready` FAILS without the final cross-family ship-gate receipt (P0); a NEW build on a legacy `ui_marker_manifest` FAILS `cx check design-fidelity` (P0 — the advisory escape is removed); a money/login/data module with no extracted-actuals conformance diff, a missing quality card (core four), or a shared-shell module with a non-PASS regression-smoke FAILS `cx check module-quality` (P1); a self-review card may run a bounded re-review escalation to a cap of 3 (cross-family stays one-shot); a post-ship CEO-found defect recorded in `open_findings` (found_post_ship + found_by) blocks final-ready. *(added: V1.10 module_acceptance fold; mechanical body = MODULE-START-* · ANDON-WALL-* · SHIPPED-WITHOUT-FINAL-CROSS-FAMILY · DESIGN-FIDELITY-SHIPPED-ADVISORY · CONFORMANCE-DRIFT-SHIPPED · QUALITY-CARD-REQUIRED · REGRESSION-SMOKE-SKIPPED · SELF-REVIEW-FIX-CYCLES-CAP-3 · CROSS-FAMILY-DEFER-PERMITS-SHIP · DEFECT-LEDGER-POST-SHIP-FOUND-BY)*
